## 🎯 Core Vision

Aiopad started as a basic floating notepad. V2 transforms it into a **multimodal AI knowledge base** — a place where everything you encounter gets understood, explained, and saved. YouTube video, research paper, image, URL, voice memo — everything becomes a smart, searchable, explainable note.

The user never has to leave their workflow. They never copy-paste into ChatGPT. The AI comes to the content.

## 🤖 NVIDIA NIM Service

**File:** `src/services/nvidia.ts`  
**Base URL:** `https://integrate.api.nvidia.com/v1`  
**Primary Model:** `nvidia/llama-3.3-nemotron-super-49b-v1`  
**Vision Model:** `nvidia/neva-22b`  

All models use the OpenAI-compatible `/chat/completions` endpoint.  
The API key is prefixed `nvapi-` and stored in `.env`.

### Functions to Build

#### 1. `callNvidia(model, messages, options?)`
Generic function that calls any NVIDIA NIM model.
- POST to `/chat/completions`
- Returns parsed response text string
- Handles errors — returns readable error string, never throws raw error

#### 2. `explainContent(content, contentType)`
Calls Nemotron Super 49B with a content-type-specific system prompt.

System prompts per content type:

**youtube:**
```
You are an expert tutor. Break this transcript into clear sections 
with headings. Explain key concepts in simple terms. End with 
## Key Takeaways and ## What To Remember sections.
```

**pdf:**
```
You are an academic assistant. Explain this document in plain English. 
Structure your response with these sections:
## What This Is About
## Key Arguments  
## Methodology
## Key Findings
## Limitations
## Why This Matters
## TL;DR
```

**image:**
```
1. Extract ALL text visible in this image exactly as written.
2. Describe what this image shows in detail.
3. If this is a diagram, chart, or screenshot — explain what it 
   represents and what insights it contains.
4. List key insights or takeaways.
```

**url:**
```
Summarize this webpage content. Cover: main argument, key points, 
important details, and what the reader should take away. 
Structure with clear headings.
```

**text:**
```
Explain this content clearly. Highlight the most important points. 
Structure with headings where appropriate.
```

Returns structured markdown string.

#### 3. `generateNoteMeta(content)`
Returns **JSON only** — no extra text, no markdown fences:
```json
{
  "title": "short descriptive title",
  "tag": "topic category",
  "summary": "one line summary of the content",
  "difficulty": "beginner|intermediate|advanced"
}
```

#### 4. `chatWithNotes(question, notesContext)`
System prompt:
```
You are a personal knowledge assistant. The user has these saved notes:

[notesContext]

Answer their question using the notes as context. 
Quote or reference specific notes when relevant. 
If the answer isn't in the notes, say so clearly.
```
Returns answer string with markdown formatting.

#### 5. `generateFlashcards(content)`
Returns **JSON array only** — no extra text:
```json
[
  { "front": "question", "back": "answer" },
  { "front": "question", "back": "answer" }
]
```
Generate 8-10 cards. Questions must be specific and testable.

## ✨ Feature 1 — YouTube Video → Smart Note

**File:** `src/services/youtube.ts` + `src/components/YoutubeInput.tsx`

### Service: `youtube.ts`

Function: `fetchYouTubeTranscript(url: string)`

1. Extract video ID from any YouTube URL format:
   - `youtube.com/watch?v=VIDEO_ID`
   - `youtu.be/VIDEO_ID`
   - `youtube.com/shorts/VIDEO_ID`

2. Fetch video metadata (title + thumbnail):
   ```
   GET https://noembed.com/embed?url={fullYoutubeUrl}
   ```
   Returns: `{ title, thumbnail_url }`

3. Fetch transcript — try in order:
   ```
   GET https://www.youtube-transcript-extractor.com/api?videoId={ID}
   ```
   Fallback:
   ```
   GET https://api.kome.ai/api/tools/youtube-transcripts?videoId={ID}
   ```

4. Return: `{ transcript, title, thumbnail, videoId }`  
   On error: `{ error: "Could not fetch transcript for this video" }`

### Component: `YoutubeInput.tsx`

- Text input field + submit button
- Placeholder: "Paste a YouTube link..."
- Styled to match existing MUI components

**Processing flow:**
1. `setLoading("Fetching transcript...")`
2. Call `fetchYouTubeTranscript(url)`
3. `setLoading("AI is analyzing the video...")`
4. Call `explainContent(transcript, 'youtube')`
5. `setLoading("Saving note...")`
6. Format note content:

```markdown
🎬 [Video Title]
🔗 [YouTube URL]
![thumbnail]([thumbnail_url])

---

[AI explanation output]
```

7. Call `generateNoteMeta()` → get title + tag + summary
8. Create new note using **existing Zustand store note creation method**
9. `clearLoading()`
10. Show success message: "Note saved!"

### Integration
- Find existing new note creation UI
- Add a **"🎬 YouTube"** button that shows `YoutubeInput`
- Do not change any existing note creation UI

## ✨ Feature 3 — Image → Extract + Explain Note

**File:** `src/services/imageAnalyzer.ts` + `src/components/ImageInput.tsx`

### Service: `imageAnalyzer.ts`

Function: `analyzeImage(file: File): Promise<string>`

1. Convert to base64:
```typescript
const base64 = await new Promise<string>((resolve) => {
  const reader = new FileReader()
  reader.onload = () => resolve((reader.result as string).split(',')[1])
  reader.readAsDataURL(file)
})
```

2. Call NVIDIA NIM vision model:
```typescript
POST https://integrate.api.nvidia.com/v1/chat/completions
{
  model: "nvidia/neva-22b",
  messages: [{
    role: "user",
    content: [
      { 
        type: "text", 
        text: "1. Extract ALL text visible in this image exactly as written. 2. Describe what this image shows in detail. 3. If this is a diagram, chart, or screenshot — explain what it represents. 4. List key insights."
      },
      { 
        type: "image_url", 
        image_url: { url: `data:image/jpeg;base64,${base64}` }
      }
    ]
  }],
  max_tokens: 1024
}
```

3. Return response text string

> **Note:** Remove the existing Tesseract.js OCR usage after this is working — NVIDIA NIM handles this better.

### Component: `ImageInput.tsx`

- Drag and drop zone OR click to upload
- Accept: PNG, JPG, JPEG, WEBP, GIF
- Show image preview (max 200px width) immediately on select
- Styled to match existing MUI

**Processing flow:**
1. Show image preview immediately
2. `setLoading("AI is reading the image...")`
3. Call `analyzeImage(file)`
4. Format note:

```markdown
🖼️ [filename]

[image rendered inline]

---

[AI analysis output]
```

5. `generateNoteMeta()` → tag + title
6. Save via existing Zustand store
7. `clearLoading()` → "Note saved!"

## ✨ Feature 4 — URL / Article → Smart Note

**File:** `src/services/urlFetcher.ts` + `src/components/UrlInput.tsx`

### Service: `urlFetcher.ts`

Function: `fetchUrlContent(url: string)`

1. Fetch via CORS proxy:
   ```
   GET https://api.allorigins.win/get?url={encodeURIComponent(url)}
   ```

2. Parse `response.contents` with `DOMParser`

3. Clean the HTML:
   - Remove: `script`, `style`, `nav`, `footer`, `header`, `aside` tags
   - Extract text from: `main`, `article`, `body` (whichever has most content)
   - Collapse multiple whitespace/newlines

4. Extract:
   - `title`: `document.title`
   - `text`: cleaned body text (truncate at 12000 chars)
   - `domain`: hostname from URL
   - `favicon`: `https://www.google.com/s2/favicons?domain={domain}`

5. Return: `{ title, text, domain, favicon }`

### Component: `UrlInput.tsx`

- Text input + submit button
- Placeholder: "Paste any URL — article, blog, GitHub, docs..."
- Styled to match existing MUI

**Processing flow:**
1. `setLoading("Fetching page...")`
2. Call `fetchUrlContent(url)`
3. `setLoading("AI is reading the page...")`
4. Call `explainContent(text, 'url')`
5. Format note:

```markdown
🌐 [Page Title]
🔗 [URL]

---

[AI explanation output]
```

6. `generateNoteMeta()` → tag + title
7. Save via existing Zustand store

## ✨ Feature 5 — Voice → Note

**File:** `src/services/voiceRecorder.ts` + `src/components/VoiceInput.tsx`

> No external dependencies — uses the Web Speech API built into Chrome/Edge

### Service: `voiceRecorder.ts`

```typescript
function createRecorder() {
  const SpeechRecognition = 
    window.SpeechRecognition || window.webkitSpeechRecognition
  
  if (!SpeechRecognition) return null

  const recognition = new SpeechRecognition()
  recognition.continuous = true
  recognition.interimResults = true
  recognition.lang = 'en-US'

  return {
    start(onInterim: (text: string) => void): void,
    // Calls onInterim with live transcript as user speaks
    
    stop(): Promise<string>
    // Stops recognition, returns final complete transcript
  }
}
```

### Component: `VoiceInput.tsx`

**States:**
- `IDLE`: mic button, "Record a voice note"
- `RECORDING`: red pulsing dot, live transcript appears in real time below button
- `PROCESSING`: "AI is structuring your note..."

**If SpeechRecognition not available:**  
Show: "Voice notes require Chrome or Edge browser"

**Processing flow:**
1. User clicks mic → start recording
2. Live transcript updates in real time below button
3. User clicks stop → `setLoading("AI is structuring your note...")`
4. Send transcript to NVIDIA NIM:
```
Clean up this voice transcription. Fix grammar and punctuation. 
Give it a clear title. Structure it into organized paragraphs 
or bullet points as appropriate. Keep all original meaning and content.
```
5. Format note:

```markdown
🎙️ Voice Note — [date time]

[AI structured output]

---
*Raw transcript: [original]*
```

6. `generateNoteMeta()` → tag
7. Save via existing Zustand store
8. `clearLoading()` → "Note saved!"

### Integration
- Add **"🎙️ Voice"** button to existing note creation area
- Nothing else changes

## ✨ Feature 7 — Chat With Your Notes

**File:** `src/services/notesContext.ts` + `src/components/NotesChat.tsx`

### Service: `notesContext.ts`

Function: `buildContext(notes: Note[], question: string): string`

**Relevance scoring:**
1. Extract meaningful words from question (filter: "the", "a", "is", "what", "how", etc.)
2. For each note, count how many question keywords appear in `content + tag + summary`
3. Sort notes by score descending
4. Take top 5 notes
5. If no notes score > 0, take 5 most recent notes

**Format context string:**
```
NOTE 1 [tag: Machine Learning] [type: pdf]:
[first 600 chars of content]

NOTE 2 [tag: React] [type: youtube]:
[first 600 chars of content]

...
```

### Component: `NotesChat.tsx`

**Layout:**
- Fixed position: `bottom: 0, left: 0, right: 0`
- Does NOT push main layout — overlays it
- Z-index above main content

**Collapsed state (always visible):**
- Single bar: `"💬 Ask your notes"` + note count + expand chevron
- Height: 48px
- Click to expand

**Expanded state:**
- Height: 360px, slides up with animation
- Header: "💬 Ask your notes" + `Searching [X] notes` + collapse button
- Message history: scrollable, with user and AI message bubbles
- AI messages: render markdown
- Input bar + Send button at bottom
- Enter key sends message

**Starter suggestions** (shown before first message, disappear after):
- "What have I saved this week?"
- "Explain my most recent note"
- "What topics have I been studying?"

**On send:**
1. Show user message immediately
2. Show typing indicator `...`
3. Call `buildContext(allNotes, question)`
4. Call `chatWithNotes(question, context)` from `nvidia.ts`
5. Show AI response with markdown rendering
6. If response mentions a note title that exists in the store, 
   make that title **bold and clickable** to open that note

### Integration
- Mount `NotesChat` once in the main app layout component
- Pass all notes from existing Zustand store as prop

## ✨ Feature 9 — Shareable Notes

### Install

### Supabase Table Schema
Create this table in the Supabase dashboard:
```sql
CREATE TABLE shared_notes (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  content TEXT NOT NULL,
  meta JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Enable public read access
ALTER TABLE shared_notes ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Public read" ON shared_notes FOR SELECT USING (true);
CREATE POLICY "Public insert" ON shared_notes FOR INSERT WITH CHECK (true);
```

### Functions

`shareNote(note: Note): Promise<string>`
1. Insert `{ content: note.content, meta: { title, tag, contentType, summary } }`
2. Return URL: `${window.location.origin}/share/${insertedId}`
